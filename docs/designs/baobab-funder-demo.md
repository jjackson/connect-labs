## Context

The Baobab Institute in West Africa is receiving a Bloomberg grant and evaluating Connect as their regranting platform. The demo needs to show the full arc: fund creation → solicitation → grantee selection → delivery monitoring — and make Baobab confident that Connect serves both funders AND their grantees.

## Vision

### 10x Check
The 10x version: Baobab logs in and sees not just *what happened* but *what to do next*. The dashboard is proactive — it surfaces risks before they become problems, writes the Bloomberg report automatically, and lets Baobab's team spend 5 minutes a week on portfolio monitoring instead of 5 hours. AI helps both sides: funders get an analyst, applicants get a coach.

### Platonic Ideal
A funder opens Connect and feels *confident*. They feel like they have a team of 10 analysts working for them. The platform levels the playing field — small orgs in rural West Africa get the same quality coaching a big NGO's grants team provides. The system knows which grantees are struggling before they report it, writes the donor report in the funder's voice, and shows human impact with faces and stories, not just numbers.

## Key Insight: Two-Sided Value

The demo narrative isn't just "Connect helps funders." It's:
- **For funders:** "AI is your analyst — it reviews applications, monitors performance, and writes your reports."
- **For applicants:** "AI is your coach — it helps you write stronger applications and become fundable."
- **For Bloomberg:** "Your money creates more impact because the platform makes both sides better."

## Framing Note: Delivery Pace (Not Burn Rate)

Spending the money IS the goal — it means impact is being delivered. Language should be positive:
- "On track to deliver full impact by August 2026" (not "fund depletes by...")
- "Delivery pace" (not "burn rate")
- "Impact delivered" (not "budget consumed")

## Accepted Scope — Technical Details

### 1. AI Narrative Report Generator ("Bloomberg Report")
**Files:** New `ai/agents/fund_report_agent.py`, modify `fund_detail.html`, extend `AIStreamView` dispatch
**Approach:** Add a "Generate Report" button on fund detail page. Client aggregates already-loaded visits/payments into summary stats via new `aggregateForReport()` JS function, then POSTs summary to `AIStreamView` with `agent=fund_report`. New pydantic-ai agent receives summary stats (not raw rows) and generates narrative. Streams via existing delta SSE pattern.
**Acceptance criteria:**
- Button on fund detail page triggers POST to AIStreamView
- Client sends aggregated summary stats (few KB), not raw visit arrays
- AI generates narrative including: fund overview, per-grantee highlights, delivery pace, recommendations
- Rendered as streaming markdown in a panel below the charts
**Depends on:** Existing `AIStreamView` pattern (POST, delta streaming, pydantic-ai). Add `fund_report` agent type to dispatch map.

### 2. Delivery Pace Forecast
**Files:** Modify `funder-charts.js` (new `renderForecast()` function), modify `fund_detail.html`
**Approach:** New Chart.js line chart. Plots actual cumulative USD distributed by week. Calculates linear regression on the last 8 weeks to project forward. Shows projected line as dashed. Displays headline: "On track to deliver full impact by {date}" with positive framing.
**Acceptance criteria:**
- Chart shows actual spend (solid line) + projected trajectory (dashed line)
- Headline calculates projected completion date from trend
- Positive language throughout ("delivery pace", "impact delivered")
- Handles edge cases: no data yet, budget already exceeded, flat trend
**Depends on:** Payment data already loaded by SSE pipeline

### 3. Cross-Chart Filtering
**Files:** Modify `funder-charts.js` (new filter bus module), modify `fund_detail.html` (filter controls UI)
**Sub-features:**
- **3a. Filter controls UI** (S) — Delivery type multi-select, country multi-select, date range picker. Rendered above charts. Alpine.js for state management.
- **3b. Filter bus** (S) — Lightweight JS event bus: filter state object, `applyFilters()` function that re-renders all charts/KPIs/map with filtered data. No React needed — each render function already accepts data arrays, so filtering happens at the data level before calling existing render functions.
- **3c. Cross-chart click filtering** (S) — Clicking a bar/segment in visits chart sets opportunity filter, map responds. Click handler on Chart.js, dispatches to filter bus.
**Acceptance criteria:**
- All 3 filter types work independently and in combination
- All charts, KPIs, alerts, and map respond to active filters
- Clear filters button resets to full view
- **Stretch:** URL params persist filter state
**Architecture note:** No React conversion needed. Filter state lives in an Alpine.js reactive object. Existing render functions are called with filtered data arrays.

### 4. Solicitation-to-Fund Cross-Link
**Files:** Modify `portfolio.html` or `fund_detail.html` (add link), modify `fund_form.html` (fund selector)
**Approach:** On portfolio page, each fund card shows "X active solicitations" with link to `/solicitations/?fund_id=Y`. Solicitations app already handles the full experience. Minimal change — just a link and a count.
**Acceptance criteria:**
- Fund card shows solicitation count if any linked solicitations exist
- Link navigates to solicitations list filtered by fund
**Data access:** Requires adding `get_solicitations_by_fund_id()` to `SolicitationsDataAccess` (uses `data__fund_id` API filter). Portfolio view must instantiate `SolicitationsDataAccess` for counts.
**Depends on:** `fund_id` field already exists on SolicitationRecord

### 5. AI-Powered Solicitation Review
**Files:** New `ai/agents/solicitation_review_agent.py`, modify response list template, extend `AIStreamView` dispatch
**Approach:** New "AI Review All" button on `ResponsesListView`. POSTs to `AIStreamView` with `agent=solicitation_review`. Agent fetches all responses via tool calls (using existing `SolicitationsDataAccess.get_responses_for_solicitation()`), reads evaluation criteria, generates comparative analysis. Streamed via existing delta SSE pattern. Does NOT replace manual per-response review — supplements it.
**Acceptance criteria:**
- Button on response list page triggers POST to AIStreamView
- AI receives all responses + evaluation criteria + solicitation details
- Output includes comparative ranking, risk flags per applicant, recommended shortlist
- Manual review flow remains unchanged
**Depends on:** Existing `AIStreamView` pattern. Add `solicitation_review` agent type to dispatch map.

### 6. AI Application Coach for Applicants
**Files:** New `ai/agents/application_coach_agent.py`, modify `solicitations/templates/respond.html` (add AI panel), extend `AIStreamView` dispatch
**Approach:** Collapsible sidebar in the response form. AI reads solicitation description, questions, AND evaluation criteria (criteria are already visible in the form UI). When applicant clicks "Get Feedback", client POSTs draft answers to `AIStreamView` with `agent=application_coach`. Agent helps applicant address criteria directly.
**Acceptance criteria:**
- Collapsible AI panel appears in response form
- AI reads solicitation description + questions + criteria (already public in form)
- Provides specific, actionable suggestions per answer
- Does not auto-submit or modify answers — advisory only
**Security note:** Must NOT expose OTHER applicants' responses. Agent receives only the current user's draft answers + solicitation data.

### 7. PDF One-Pager Export
**Files:** New `funder_dashboard/pdf_export.py`, new template `funder_dashboard/report_pdf.html`, new view in `views.py`
**Approach:** Django template rendered to HTML, converted to PDF via ReportLab or browser print CSS. Avoid weasyprint (requires system-level Cairo/Pango on ECS Fargate). Alternative: client-side PDF via `window.print()` with a print-optimized CSS stylesheet — simplest approach, no new dependencies.
**Acceptance criteria:**
- "Download Report" button on fund detail page
- PDF includes: fund name/description, KPI summary, allocation table, AI narrative (if generated)
- Clean, branded layout suitable for board meetings
**Infrastructure note:** Prefer client-side `window.print()` + print CSS to avoid adding server dependencies. If richer formatting needed, ReportLab (pure Python, already pip-installable) over weasyprint.

## Eng Review Decisions (2026-03-22)

1. **AI Streaming:** All 3 AI features use `AIStreamView` (POST, delta streaming) — not `BaseSSEStreamView` (which is for data pipelines). Add `fund_report`, `solicitation_review`, `application_coach` to the agent dispatch map.
2. **AI Report Data Flow:** Client aggregates already-loaded visits/payments into summary stats, POSTs to AIStreamView. No re-fetching from production API.
3. **AI Coach Security:** Coach CAN see evaluation criteria (already visible in form UI). Real security boundary: never expose other applicants' responses.
4. **AI Testing:** Mock AI for CI tests (verify inputs, streaming, security boundaries). Real AI testing during dev/demo prep.
5. **PDF Print Gap:** Chart.js `<canvas>` elements may not render in `window.print()`. Use `Chart.toBase64Image()` to convert canvases to `<img>` tags before printing.
